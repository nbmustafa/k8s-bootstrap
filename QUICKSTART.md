# 🚀 Quick Start Guide

Get your Kubernetes sandbox up and running in 5 minutes!

## Prerequisites Check

```bash
# Verify you have all required tools
docker --version    # Should be 20.10+
kubectl version     # Should be 1.28+
helm version       # Should be 3.12+
kind version       # Should be 0.20+
```

**Don't have these installed?** See the [main README](README.md#prerequisites) for installation instructions.

## Option 1: Standard Bootstrap (Recommended for First Time)

This is the simplest way to get started:

```bash
# 1. Run the bootstrap script
./bootstrap.sh

# That's it! Wait 5-10 minutes for everything to deploy.
```

## Option 2: GitOps Bootstrap (Recommended for Learning GitOps)

This version creates a local git repository for true GitOps workflow:

```bash
# 1. Run the GitOps bootstrap script
./bootstrap-gitops.sh

# 2. After deployment, you can make changes to the gitops-repo/ directory
# 3. Commit changes and ArgoCD will auto-sync them
```

## Option 3: Using Make (Most Convenient)

```bash
# Deploy everything
make bootstrap

# View status
make status

# Open dashboards
make dashboard

# Clean up
make clean
```

## What Happens During Bootstrap?

1. **Checks Prerequisites** - Verifies Docker, kubectl, Helm, Kind are installed
2. **Downloads Helm Charts** - Pulls ArgoCD, Prometheus, Grafana charts locally
3. **Creates Kind Cluster** - Spins up a local Kubernetes cluster
4. **Deploys ArgoCD** - Installs GitOps engine
5. **Deploys Monitoring** - Prometheus & Grafana via Helm
6. **Exposes Services** - Maps ports to localhost

kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d

make info

## First Steps After Bootstrap

### 1. Explore ArgoCD

```bash
# Open ArgoCD UI
open http://localhost:8080

# Or click here: http://localhost:8080
```

**What to check:**
- ✅ Applications are "Synced" and "Healthy"
- ✅ All resources are green
- 🔍 Click on "prometheus-stack" to see the deployment tree

### 2. Explore Prometheus

```bash
# Open Prometheus UI
open http://localhost:9090

# Try these queries:
# - up                                    (see all targets)
# - node_cpu_seconds_total               (CPU metrics)
# - kube_pod_status_phase{phase="Running"} (running pods)
```

### 3. Explore Grafana

```bash
# Open Grafana UI
open http://localhost:3000

# Login: admin / admin
```

**Pre-installed Dashboards:**
- Kubernetes Cluster Overview
- Node Exporter Full
- Prometheus Stats

Navigate: Home → Dashboards → Browse

### 4. Check Everything is Running

```bash
# View all pods
kubectl get pods -A

# Should see:
# - argocd namespace: ~7 pods
# - monitoring namespace: ~15+ pods
# - All should be Running

# Quick validation
make test
```

## Common First-Time Issues

### Issue: Bootstrap script fails with "Docker not running"

**Solution:**
```bash
# Start Docker Desktop or Docker daemon
# macOS: Open Docker Desktop app
# Linux: sudo systemctl start docker
```

### Issue: Pods stuck in "Pending" state

**Solution:**
```bash
# Check pod details
kubectl describe pod <pod-name> -n <namespace>

# Common cause: Insufficient resources
# Solution: Increase Docker Desktop resources
# Settings → Resources → Increase CPUs to 4, Memory to 8GB
```

### Issue: Can't access services at localhost

**Solution:**
```bash
# Verify port mappings
docker ps

# Should see ports like:
# 0.0.0.0:8080->30080/tcp
# 0.0.0.0:9090->30081/tcp
# 0.0.0.0:3000->30082/tcp
```

### Issue: Helm charts fail to download

**Solution:**
```bash
# Update Helm repos manually
helm repo update

# Then re-run bootstrap
./bootstrap.sh
```

## Next Steps

Once everything is running:

1. **Customize Grafana Dashboards**
   - Import dashboards from https://grafana.com/grafana/dashboards/
   - Edit `helm-charts/prometheus/values.yaml` to add more dashboards

2. **Set Up Alerting Rules**
   - Create custom Prometheus alerts
   - Configure Alertmanager notifications

3. **Deploy Your Own Apps**
   - Create ArgoCD Applications
   - Deploy via GitOps workflow

4. **Learn GitOps**
   - Use `bootstrap-gitops.sh` version
   - Make changes in `gitops-repo/` directory
   - Watch ArgoCD auto-sync

## Useful Commands

```bash
# View cluster info
kubectl cluster-info

# Get all resources in monitoring namespace
kubectl get all -n monitoring

# View logs
kubectl logs -n monitoring <pod-name>

# Execute command in pod
kubectl exec -it -n monitoring <pod-name> -- /bin/sh

# Port forward a service
kubectl port-forward -n monitoring svc/prometheus-kube-prometheus-prometheus 9090:9090

# Access ArgoCD CLI
kubectl port-forward -n argocd svc/argocd-server 8080:443
argocd login localhost:8080 --username admin --password <password> --insecure
```

## Cleanup

When you're done:

```bash
# Option 1: Use cleanup script
./cleanup.sh

# Option 2: Use make
make clean

# Option 3: Manual
kind delete cluster --name sandbox-cluster
```
